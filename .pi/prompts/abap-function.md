# /abap-function — ABAP 函数模块开发与修改

## 重要限制
**Agent 不支持创建函数模块**。函数模块必须由用户在 SE37 中手动创建（含接口参数定义）。Agent 只能：
- 读取已有函数模块的源码和接口参数
- 修改函数模块的 ABAP 代码体（不含 `*"` 签名注释块）
- 写入 SAP 并激活

## 触发条件
- 用户需要开发一个新的函数模块 → 用户 SE37 创建空壳 → Agent 写代码
- 用户需要修改已有函数模块的代码

## 前置阅读
- 编码规范：`SYSTEM.md`
- 代码模板与最佳实践：`.pi/skills/abap-function/SKILL.md`
- 工具用法：`.pi/skills/gxx-abap/SKILL.md`

## 工作流程（含确认门禁，⚠ 处必须等用户明确确认）

### 1. 理解需求
- 是新建还是修改？函数模块名是什么？
- 需要哪些输入参数？输出参数？业务逻辑做什么？

信息不全 → 直接向用户提问，不得自行假设。

### 2. 确认函数模块名
⚠ 用户确认函数模块名后才能继续。

### 3. 新建 → 用户创建空壳
如果函数模块不存在：
说明"Agent 不支持创建函数模块，请在 SE37 中创建，定义好接口参数后告诉我"。
```
在 SE37 中：
1. 输入函数模块名
2. 定义 IMPORTING / EXPORTING / TABLES / EXCEPTIONS
3. 保存激活
```

### 4. 读取源码获取接口参数
```
abap_ls -pattern "<函数模块名>"       # 确认存在
abap_cat -path <函数模块名> -t fm     # 读取源码
```
从 `*"` 注释块提取接口参数。向用户展示：
```
函数：<函数模块名>
IMPORTING：<参数清单>
EXPORTING：<参数清单>
TABLES：<参数清单>
EXCEPTIONS：<异常清单>
```

### 5. 确认表结构和逻辑（输出契约 A）
```
abap_meta -name <表名>
```
按以下格式汇报，⚠ 用户确认后再写代码：
```
处理逻辑：
1. 输入校验：<校验逻辑>
2. 数据读取：<查什么表>
3. 业务处理：<核心逻辑>
4. 结果返回：<返回什么>
请确认是否有需要修改的地方？
```

### 6. 生成代码并展示
遵循 `SYSTEM.md` 规范，生成 ABAP 代码体（**不写** `*"` 接口签名注释块），写入 `output/<函数模块名>.abap`，全文展示。
⚠ 用户确认后才能写入 SAP。

### 7. 写入 SAP
gxx-abap 会自动检测传输号，无需手动指定 --transport。
```
abap_put -path <函数模块名> -file output/<函数模块名>.abap -t fm
abap_activate -path <函数模块名> -t fm
```
激活报错必须修复，同步更新本地文件。同一问题连续失败 3 次 → 停手上报。

## 输出契约 B（完成后汇报）
```
已完成：
- 函数模块：<函数名>
- 本地文件：output/<函数模块名>.abap
- 状态：已激活
```
